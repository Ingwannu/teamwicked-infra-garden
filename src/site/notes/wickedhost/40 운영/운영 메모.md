# 운영 메모

이 문서는 현재 TeamWicked 인프라의 운영 원칙을 모아둔 런북입니다. 장애 상황에서는 반드시 [[wickedhost/70 장애대응/장애 대응 체크리스트|장애 대응 체크리스트]]와 함께 봅니다.

## WireGuard 팬아웃 운영 규칙

- 운영 경로는 항상 `WireGuard`
- 운영 프로필
  - `WireGuard only`
  - `MTU = 1280`
  - `UDP port = 20080`
- `FOU/IPIP`는 fallback 전용이며 평시 비활성 유지
- 팬아웃 허브는 [[wickedhost/20 서버/ingwprox1|ingwprox1]]

자세한 IP 구성은 [[wickedhost/10 인프라/인프라 개요|인프라 개요]]와 [[wickedhost/60 DNS/Cloudflare DNS 정리|Cloudflare DNS 정리]]를 봅니다.

## DNS 운영 규칙

- `node*`, `snode*`는 팬아웃 공인 IP에 고정
- 백엔드 실IP 변경은 `theireal*`만 갱신
- 앱 도메인 이전 시 Cloudflare 프록시 여부와 origin 인증서를 같이 확인

## Pterodactyl 노드 운영 메모

- 패널에서 관리하는 주요 노드 FQDN은 현재 아래 기준
  - `snode1.teamwicked.me`
  - `snode2.teamwicked.me`
  - `snode3.teamwicked.me`
  - `smaxnode1.teamwicked.me`
- `node*`, `snode*`는 운영용 프록시/Fanout 도메인이고, backend DDNS가 건드리면 안 됨
- 패널과 Wings 통신은 `https` 기준으로 보며, `401`은 노드 API 생존 확인에 유효한 기준이 될 수 있음
- 노드 FQDN을 수정할 때는 아래를 같이 맞춰야 함
  - 패널에 저장된 FQDN
  - Wings 인증서 SAN/CN
  - 실제 Cloudflare/origin DNS
  - 필요 시 `Behind Proxy` 여부
- 노드 응답이 느릴 때는 패널 웹 자체와 노드 API timeout을 분리해서 봐야 함
  - 특히 `snode*` 응답 지연 때문에 패널 전체가 느려 보일 수 있음

## Pterodactyl 서브도메인 운영 메모

- Blueprint `subdomainmanager`는 allocation `ip_alias`를 기준으로 DNS를 만듦
- 현재 운영 기준으로는
  - alias가 도메인이면 `CNAME`
  - alias가 IP면 `A`
  로 만드는 방식이 맞음
- 이유:
  - `node*` / `maxnode*` 같은 운영 alias를 계속 따라가게 해야 프록시 IP 변경/교체 시 기존 서브도메인도 같이 살아남음
- 과거에는 일부 레코드가 `real IP`로 고정돼서 문제가 있었고, 이를 Cloudflare에서 재생성해 교정함
- subdomain 관련 이상 시에는 패널 DB보다 먼저 Cloudflare 실제 레코드 타입과 content를 확인하는 것이 빠름

## Blueprint 익스텐션 운영 메모

- 현재 패널에는 Blueprint 익스텐션이 다수 설치돼 있음
- 운영상 특히 많이 건드리는 범주는 아래와 같음
  - 노드/도메인: `subdomainmanager`
  - 로그인/계정: `sociallogin`
  - Minecraft 서버 편의: `mcplugins`, `mcmods`, `modpackinstaller`, `minecraftplayermanager`, `versionchanger`, `mclogs`
  - UI/테마: `nebula`, `loader`, `simplefavicons`, `simplefooters`, `snowflakes`, `nightadmin`
  - 파일/리소스/자동화: `filesearch`, `advancedmovefiles`, `resourcealerts`, `resourcemanager`, `sysautomation`, `consolelogs`
- Blueprint 관련 설정은 `settings` 테이블에 들어가며, 키 형식은 대체로 아래 패턴
  - `blueprint::...`
  - `<extension>::...`
- 패널 UI가 이상하게 바뀌거나 일부 메뉴가 사라지면, 패널 본체보다 Blueprint 확장 영향부터 의심하는 것이 빠름
- 패널 캐시를 비워도 바로 안 바뀌면
  - 확장 설정값
  - 관련 라우트 노출 여부
  - frontend wrapper
  를 같이 봐야 함
- TeamWicked 환경에선 특히 `subdomainmanager`가 프록시/Fanout 구조와 직접 맞물리므로, 이 확장 동작을 일반적인 단독 호스팅 환경처럼 보면 안 됨
