# 운영 메모

이 문서는 현재 TeamWicked 인프라의 운영 원칙을 모아둔 런북입니다. 장애 상황에서는 반드시 [[wickedhost/70 장애대응/장애 대응 체크리스트|장애 대응 체크리스트]]와 함께 봅니다.

## WireGuard 팬아웃 운영 규칙

- 운영 경로는 항상 `WireGuard`
- 운영 프로필
  - `WireGuard only`
  - `MTU = 1280`
  - `UDP port = 20080`
- `FOU/IPIP`는 fallback 전용이며 평시 비활성 유지
- 팬아웃 허브는 [[wickedhost/20 서버/ingwprox1|ingwprox1]]

자세한 IP 구성은 [[wickedhost/10 인프라/인프라 개요|인프라 개요]]와 [[wickedhost/60 DNS/Cloudflare DNS 정리|Cloudflare DNS 정리]]를 봅니다.

## DNS 운영 규칙

- `node*`, `snode*`는 팬아웃 공인 IP에 고정
- 백엔드 실IP 변경은 `theireal*`만 갱신
- 앱 도메인 이전 시 Cloudflare 프록시 여부와 origin 인증서를 같이 확인

관련 문서:
- [[wickedhost/30 서비스/서비스 및 도메인|서비스 및 도메인]]
- [[wickedhost/60 DNS/Cloudflare DNS 정리|Cloudflare DNS 정리]]

## WickedShip 운영 규칙

- 현재 운영 호스트는 [[wickedhost/20 서버/ingwprox2|ingwprox2]]
- `devpanel2.teamwicked.me`는 현재 이 호스트를 가리킴
- ARM 이전 후 [[wickedhost/20 서버/ingwprox4|ingwprox4]]는 주 엔드포인트가 아님
- 신규 app/control-plane VM은 초기 bootstrap 때 호스트 방화벽을 지나치게 조이지 말 것

- `garden.teamwicked.me`는 GitHub Pages가 아니라 [[wickedhost/20 서버/ingwprox1|ingwprox1]]에서 직접 호스팅

- 사이트 업데이트는 이 서버에서 `garden update`
- 문서 퍼블리시는 현재 `wickedhost` 아래 모든 `.md`를 대상으로 함
