# 운영 메모

## WireGuard 팬아웃 운영 규칙

- 운영 경로는 항상 `WireGuard`
- 운영 프로필:
  - `WireGuard only`
  - `MTU = 1280`
  - `UDP port = 20080`
- `FOU/IPIP`는 fallback 전용이며 평시 비활성 유지

## DNS 운영 규칙

- `node*`, `snode*`는 팬아웃 공인 IP로 고정
- 백엔드 실IP 변경은 `theireal*`만 갱신

## WickedShip 운영 규칙

- 현재 운영 호스트는 `ingwprox2`
- ARM 이전 후 `ingwprox4`는 주 엔드포인트가 아님
- 신규 app/control-plane VM은 초기 부트스트랩 시 호스트 방화벽을 지나치게 조이지 말 것

## Grafana / Blesta 메모

- Grafana는 `ingwprox1`에 있음
- Blesta는 `103.214.68.149`에 남아 있음
- 두 서비스를 같은 머신으로 취급하면 안 됨

## SSH 키 메모

Oracle 앱 서버(`ingwprox2`, `ingwprox3`)에는 `ubuntu:~/.ssh/authorized_keys`에 아래 두 키를 유지:
- 기존 TeamWicked ingw 키
- dFlow 공개키

## 추후 추가하면 좋은 문서

- Oracle 비용 추적
- Cloudflare DNS 운영 표
- WickedShip / k3s 운영 절차
- 장애 대응 체크리스트
