# 장애 대응 체크리스트

## 팬아웃 장애

1. Oracle에서 `10.200.1.2`, `10.200.2.2`, `10.200.3.2` 핑 확인
2. 각 백엔드에서 Oracle `10.200.x.1` 역방향 핑 확인
3. `WireGuard` 상태와 `MTU 1280` 유지 여부 확인
4. `FOU/IPIP`는 복구 테스트가 아니면 켜지 말 것

## WickedShip 장애

1. `devpanel2.teamwicked.me` HTTPS 응답 확인
2. `payload-app` / `mongodb` / `redis` / `devpanel2-nginx` 상태 확인
3. 최신 이미지 반영 시 `docker pull`과 `compose up` 이후 앱 내부 `3000` 리슨 확인
4. ARM 전환 후 `ingwprox2`가 현재 운영 호스트임을 전제할 것

## Grafana 장애

1. `https://dashb.teamwicked.me/login` 응답 확인
2. `127.0.0.1:15433` PostgreSQL 브리지 리슨 확인
3. `pmacct-postgres`와 `pmacctd` 적재 상태 확인
4. collector가 old host가 아니라 로컬 DB를 보도록 설정돼 있는지 확인

## 노드/클러스터 장애

1. control-plane은 현재 `ingwprox4` 대신 앱 구조상 실제 활성 호스트를 먼저 확인
2. `k3s kubectl get nodes -o wide`로 실제 조인 상태 확인
3. UI 미반영과 실제 k3s 조인을 혼동하지 말 것
4. Oracle 사설망 포트 reachability와 호스트 방화벽을 함께 확인할 것
