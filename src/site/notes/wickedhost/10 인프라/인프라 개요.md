# 인프라 개요

현재 TeamWicked 인프라는 Oracle 기반 프론트 허브 + 앱 서버 + 별도 Blesta 서버 구조입니다. 상세 서버 목록은 [[wickedhost/20 서버/서버 인벤토리|서버 인벤토리]], 도메인 연결은 [[wickedhost/30 서비스/서비스 및 도메인|서비스 및 도메인]], 운영 규칙은 [[wickedhost/40 운영/운영 메모|운영 메모]]를 봅니다.

## 전체 구조

- [[wickedhost/20 서버/ingwprox1|ingwprox1]]
  - Oracle ARM 프론트 허브 및 관리 서버
  - WireGuard 팬아웃 허브
  - Grafana 및 pmacct PostgreSQL 브리지 호스트
- [[wickedhost/20 서버/ingwprox2|ingwprox2]]
  - Oracle ARM 앱 서버
  - 현재 [[wickedhost/30 서비스/WickedShip 운영|WickedShip 운영]] 호스트
  - `devpanel2.teamwicked.me`
- [[wickedhost/20 서버/ingwprox3|ingwprox3]]
  - Oracle x86 앱 서버
  - 현재 [[wickedhost/30 서비스/dFlow 운영|dFlow 운영]] 호스트
  - `devpanel1.teamwicked.me`
  - `worker-seoul-01` k3s 워커 역할
- [[wickedhost/20 서버/ingwprox4|ingwprox4]]

- [[wickedhost/20 서버/Blesta 서버|Blesta 서버]]

## 팬아웃 구조

- 운영 전송 경로: `WireGuard`
- 프론트 허브: [[wickedhost/20 서버/ingwprox1|ingwprox1]]
- Oracle `wg0`
  - `ListenPort = 20080/udp`
  - `MTU = 1280`
- 팬아웃 공인 IP
  - `152.70.94.228 -> node1`
  - `146.56.153.18 -> node2`
  - `146.56.157.95 -> node3`
- 백엔드 실IP
  - `node1 -> 59.13.221.134`
  - `node2 -> 121.157.38.232`
  - `node3 -> 220.78.127.66`

이 규칙의 상세 운영 원칙은 [[wickedhost/40 운영/운영 메모|운영 메모]]와 [[wickedhost/70 장애대응/장애 대응 체크리스트|장애 대응 체크리스트]]에 정리합니다.

## 클러스터 상태

- 현재 WickedShip 제어면은 [[wickedhost/20 서버/ingwprox2|ingwprox2]]에서 운영 중입니다.
- 별도 노드 연결/클러스터 실험은 [[wickedhost/20 서버/ingwprox3|ingwprox3]] 쪽 x86 서버를 중심으로 했습니다.
- 과거 [[wickedhost/20 서버/ingwprox4|ingwprox4]]는 control-plane / WickedShip 호스트로 사용했지만 현재는 주 엔드포인트가 아닙니다.

## 핵심 운영 포인트

- `FOU/IPIP`는 복구용 대기 경로이며 운영 경로가 아닙니다.
- `WireGuard only + MTU 1280 + UDP 20080`이 현재 안전 프로필입니다.
- `node*`, `snode*`는 팬아웃 공인 IP에 고정해야 합니다.
- `theireal1/2/3.teamwicked.me`만 DDNS 대상입니다.
- 앱 호스트 이전이나 도메인 이동 시 [[wickedhost/60 DNS/Cloudflare DNS 정리|Cloudflare DNS 정리]]와 [[wickedhost/70 장애대응/장애 대응 체크리스트|장애 대응 체크리스트]]를 함께 봅니다.
